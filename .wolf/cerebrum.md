- [2026-05-05] Do not use single-method process detection (e.g., only lsof) for concurrent safety checks — if the tool is unavailable, the check silently fails and data corruption results. Use multiple methods (/proc, lsof, fuser) with fallback to conservative error. (bugs 017)
- [2026-05-05] Do not directly access sparse associative arrays without validation (e.g., `${arr[$id]}` when some IDs may not be in the array). Validate with `${arr[$id]:-}` and handle empty case. Use helper functions to enforce consistency. (bug-018)

- [2026-05-07] Do not let `find_handoff_for` fall back to a cross-epic glob (`docs/roadmap/*/session-NN-handoff.md`) when the current-epic probe fails. The DAG dependency model is intra-epic by construction (`depends_on:` takes session ids within the current epic), so the fallback's only practical firing condition is "parent in the SAME epic produced no handoff" — and it silently grabs the alphabetically-first OTHER epic's handoff, framing it in the prompt as "the only memory of prior work" via build_handoff_section's header. - [2026-05-07] Do not decode chunked stdin via per-chunk `bytes.decode("utf-8", errors="replace")` when the producer can emit multi-byte UTF-8 (em-dashes, smart quotes, accented filenames). `read1(65536)` returns whatever the pipe has buffered, so a 2/3/4-byte codepoint can land split across two chunks; per-chunk decoding treats each fragment as its own complete string and the `errors="replace"` policy substitutes `�` on BOTH sides of the split. Audit class: any streaming text consumer that calls `bytes.decode(...)` per read should be using an incremental decoder. (bug-162)

[` glob meta gets substituted with whatever cwd happens to match, and any path with literal whitespace splits into bogus fragments. Both classes corrupt the very diagnostic the unquoted form was trying to print. Always iterate via `while IFS= read -r f; do printf '  %s\n' "$f"; done <<< "$VAR"` — same pattern `auto_resolve_wolf_conflicts` already uses. (bug-163)
